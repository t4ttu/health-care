# 健康管理アプリ 要件定義書

**バージョン:** 1.0  
**作成日:** 2026-05-06  
**ステータス:** ドラフト

---

## 1. プロジェクト概要

### 1.1 目的
自分自身の体重・体脂肪などの身体状態と運動習慣を継続的に記録・可視化することで、健康管理の意識向上と習慣化を支援するWebアプリケーションを個人開発する。

### 1.2 ターゲットユーザー
- 主ユーザー：開発者本人（個人利用）
- 将来的な拡張：複数ユーザー対応も視野に入れた設計とする

### 1.3 スコープ（今回の開発範囲）
| カテゴリ | 内容 |
|----------|------|
| 身体状態管理 | 体重・体脂肪率の記録・グラフ表示 |
| 運動管理 | 運動種目・時間・消費カロリーの記録・集計 |
| 認証 | サインイン / サインアップ（NextAuth.js） |

> 食事・カロリー管理は将来バージョン（v2）での対応とする。

## 3. 機能要件

### 3.1 認証

| # | 機能 | 優先度 |
|---|------|--------|
| A-1 | メールアドレス + パスワードでのサインアップ | Must |
| A-2 | サインイン / サインアウト | Must |
| A-3 | 未認証ユーザーのルートガード（ミドルウェア） | Must |

> パスワードリセット（メール送信による再設定）は v1 対象外とする（v1.1 以降で検討）。

#### 認証の詳細要件
| # | 要件 | 優先度 |
|---|------|--------|
| A-4 | users.email は一意制約を必須とする | Must |
| A-5 | パスワードはハッシュ化して保存し、平文保存を禁止する | Must |
| A-6 | パスワードは 8 文字以上を必須とする | Must |
| A-7 | サインアップ時に重複メールアドレスはエラー表示する | Must |
| A-8 | 連続ログイン失敗時のレート制限を行う（5回失敗で10分ロック。詳細は設計で定義） | Should |

---

### 3.2 身体状態管理

#### 記録できるデータ項目
| フィールド | 型 | 必須 | 備考 |
|------------|----|------|------|
| 記録日 | date | ✅ | デフォルト：当日 |
| 体重 (kg) | decimal | ✅ | 小数点第1位まで |
| 体脂肪率 (%) | decimal | ❌ | 任意入力 |
| メモ | text | ❌ | 自由記述 |

#### 同一日付の記録について

| 対象 | ポリシー |
|------|----------|
| body_logs | 1日1件とする（同一 user_id + recorded_at の重複登録はエラー。UNIQUE制約を付与） |
| exercise_logs | 1日複数件可（種目が異なる運動を複数記録できる） |


#### 機能一覧
| # | 機能 | 優先度 |
|---|------|--------|
| B-1 | 身体データの新規記録 | Must |
| B-2 | 記録の編集・削除 | Must |
| B-3 | 記録一覧の表示（日付降順） | Must |
| B-4 | 体重推移の折れ線グラフ（週・月単位） | Must |
| B-5 | 体脂肪率推移グラフ | Should |
| B-6 | 目標体重の設定とグラフ上への表示 | Should |
| B-7 | BMI の自動計算・表示（身長を初期設定で入力） | Want |

---

### 3.3 運動管理

#### 記録できるデータ項目
| フィールド | 型 | 必須 | 備考 |
|------------|----|------|------|
| 記録日 | date | ✅ | デフォルト：当日 |
| 運動種別ID | uuid | ✅ | 運動種別マスタ（exercise_types）を参照 |
| 時間（分） | integer | ✅ | |
| 消費カロリー（kcal） | integer | ❌ | 手入力 or 自動推算 |
| 強度 | enum | ❌ | 低・中・高 |
| メモ | text | ❌ | |

#### 運動種別マスタ（初期値）
ランニング、ウォーキング、筋トレ、水泳、サイクリング、ヨガ、その他

#### 運動種別の管理要件
| # | 要件 | 優先度 |
|---|------|--------|
| C-0-1 | 運動ログ登録時は運動種別マスタからの選択を必須とする | Must |
| C-0-2 | 運動ログは exercise_type_id で運動種別マスタを参照する | Must |
| C-0-3 | 運動種別は表示順（sort_order）で選択肢を表示する | Should |
| C-0-4 | 運動種別の有効/無効（is_active）を管理し、無効は新規選択不可とする | Should |

#### 機能一覧
| # | 機能 | 優先度 |
|---|------|--------|
| C-1 | 運動ログの新規記録 | Must |
| C-2 | 記録の編集・削除 | Must |
| C-3 | 記録一覧の表示（日付降順） | Must |
| C-4 | 週次・月次の運動時間合計グラフ（棒グラフ） | Must |
| C-5 | 消費カロリーの週次・月次集計 | Should |
| C-6 | 運動種目別の割合表示（円グラフ） | Want |
| C-7 | 連続記録日数（ストリーク）の表示 | Want |

### 3.5 集計・日付境界の定義

| 項目 | 定義 |
|------|------|
| タイムゾーン | JST（Asia/Tokyo）固定 |
| 週次集計の起点 | 月曜 00:00 開始 |
| 月次集計の起点 | 毎月 1 日 00:00 開始 |
| 日付境界 | JST の 00:00 で日付切り替え |
| 記録日の既定値 | 当日（JST） |

---

### 3.6 データアクセス制御

| # | 要件 | 優先度 |
|---|------|--------|
| E-1 | 一覧取得は user_id 一致データのみ返却する | Must |
| E-2 | 詳細取得は user_id 一致を必須条件とする | Must |
| E-3 | 更新は user_id 一致データのみ許可する | Must |
| E-4 | 削除は user_id 一致データのみ許可する | Must |
| E-5 | 他ユーザーデータへのアクセスは常に認可エラーを返す | Must |

---

### 3.7 削除・更新の運用方針

| 項目 | 方針 |
|------|------|
| 削除方式 | v1 は物理削除 |
| 削除確認 | 削除前に確認ダイアログを表示 |
| 監査ログ | v1 では対象外（v1.1 以降で検討） |
| 更新履歴 | v1 では対象外（v1.1 以降で検討） |

---

### 3.8 消費カロリーの扱い

| 項目 | 方針 |
|------|------|
| v1 の入力方式 | 手入力を正とする |
| 自動推算 | v1 では実装しない（将来バージョンで対応） |
| 集計対象 | 入力済み calories_burned のみを集計 |

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンシブ対応 | スマートフォン・PC どちらでも使用可能 |
| 認証 | ログイン必須（未認証でのデータアクセス不可） |
| データ所有 | ユーザーは自分のデータのみ参照・操作可能（Row-level isolation） |
| パフォーマンス | ダッシュボードの初回表示 3秒以内（計測条件は 4.1 を参照） |
| デプロイ | Vercel 本番環境・自動デプロイ（main ブランチ push 時） |
| 環境分離 | Neon のブランチ機能でステージング DB を分離 |

### 4.1 パフォーマンス計測条件

| 項目 | 条件 |
|------|------|
| 対象画面 | ダッシュボード (`/`) |
| 計測指標 | LCP |
| 合格基準 | LCP 3.0 秒以内 |
| データ件数前提 | body_logs 365 件、exercise_logs 365 件 |
| 計測環境 | 本番相当環境（Vercel Preview 以上） |

## 6. データモデル（概要）

```
users
  id, email, password_hash, height_cm, created_at, updated_at

exercise_types
  id, name, sort_order, is_active, created_at

body_logs
  id, user_id, recorded_at, weight_kg, body_fat_pct, memo, created_at, updated_at

exercise_logs
  id, user_id, recorded_at, exercise_type_id, duration_min,
  calories_burned, intensity, memo, created_at, updated_at
```

### 6.1 制約・インデックス

| 対象 | ルール |
|------|--------|
| users.email | UNIQUE 制約 |
| exercise_types.name | UNIQUE 制約 |
| body_logs | UNIQUE 制約 (user_id, recorded_at)（1日1件制限）|
| body_logs.user_id | NOT NULL、users.id への外部キー |
| exercise_logs.user_id | NOT NULL、users.id への外部キー |
| exercise_logs.exercise_type_id | NOT NULL、exercise_types.id への外部キー |
| body_logs | インデックス (user_id, recorded_at DESC) |
| exercise_logs | インデックス (user_id, recorded_at DESC) |
| exercise_types | インデックス (is_active, sort_order) |

### 6.2 カラム運用ルール

| カラム | ルール |
|--------|--------|
| created_at | 作成時に自動設定 |
| updated_at | 更新時に自動更新（users/body_logs/exercise_logs に追加） |
